# 入场决策 (Stock Entry Decision)

## R — 原文 (Reading)

> 你应该先运用基础分析,选择最有希望的股票。在这个基础上,再看这只股票是否在上升阶段。在具体买卖时,在突破临界点时入场,以便提高获胜概率。
>
> — 陈江挺,《炒股的智慧》第四章 何时买股票

> 我发觉新手用很多心思研究单独股票的基础层面和技术层面,认为再好的市场也有股票跌,再坏的市场也有股票升,所以忽视大市的走向。我要在这里强调: 炒股是概率的游戏,逆大潮流而动,你的获胜概率就被大打折扣了。
>
> — 陈江挺,《炒股的智慧》第二章 第三节

## I — 方法论骨架 (Interpretation)

入场决策的核心不是"选对股票"或"选对价格",而是**"三层过滤"——只有当三层都通过时才入场**。每一层都是一个独立过滤器,跳过任何一层都会显著降低胜算。

**第一层: 基础分析**——这家公司"为什么值得投"?看盈利增长、新产品/回购、行业地位等"催化剂"。这一层过滤掉"无故事可讲"的公司。

**第二层: 阶段判断**——这只股票当前处于"临界点四阶段"(牛皮/正常升势/疯狂/最后)中的哪一阶段?作者只买"正常升势"或即将突破牛皮的股票,不买疯狂阶段、不买最后阶段。这一层过滤掉"逆势操作"。

**第三层: 临界点**——入场时机是否在临界点(突破点、支撑/阻力线突破、交易量确认)?不在临界点入场意味着胜算低。这一层过滤掉"模糊时机"。

**隐含的第四层(由作者反复强调)**: 大市(牛/熊)是最高层过滤器,熊市不买任何股票。即使个股研究再充分,在熊市中胜算仍被市场β拖累。

**关键洞察**: 决策时序是**先定止损位,再定入场点**。这反直觉但关键——"我亏得起多少"决定了"我该买多少仓位",反向推导出"我应该在什么价位入场"。

## A1 — 书中的应用 (Past Application)

### 案例 1: 微软上市初期的入场决策
- **问题**: 微软上市初期市盈率超过 100,常识认为太高
- **方法论的使用**: 作者强调"成本收益比率超过 100"=高估值,但盈利增长率对应高位也是合理的——基础分析通过(新产品+高增长),阶段判断通过(早期上升),临界点(初始公开发行)出现
- **结论**: 13 年前 1 万美元的微软股票今天=百万富翁
- **结果**: 验证了"基础分析+临界点入场"的双层过滤有效

### 案例 2: 麦当劳快餐概念成形期
- **问题**: 1967-1971 年麦当劳股价翻 11 倍,但当时很多投资者不敢"高位接盘"
- **方法论的使用**: 投资者正确应用"新概念+临界点"判断——快餐作为新商业模式,正在被市场重新评估,临界点+盈利增长共振
- **结论**: 4 年 11 倍涨幅
- **结果**: 验证了"新概念+临界点"组合的爆发力

### 案例 3: 王安电脑的"绩优股幻觉"
- **问题**: 王安电脑曾是美国"绩优股""强势股"代表,作者交大校友为之骄傲
- **方法论的使用**: 表面通过基础分析(绩优),但作者没意识到阶段判断(产品周期末期)和临界点(突破已结束)已不支持入场
- **结论**: 王安电脑破产
- **结果**: 警示——基础分析通过但阶段/临界点判断错误,仍是失败入场

## A2 — 触发场景 (Future Trigger) ★

### 用户会在什么情境下需要这个 skill?

1. **候选股票已确定**: "我看好了 X 股票,你觉得现在能买吗"
2. **价格已上涨后**: "X 股票已经涨了 20%,是不是高了,现在买还来得及吗"
3. **朋友推荐后**: "我朋友推荐了 X,说一定会涨,我能跟着买吗"
4. **市场情绪好时**: "现在身边人都在赚钱,我也想入场"
5. **亏损想加仓**: "我已经亏了 30%,要不要加仓把成本降下来"(这种情况**不要**用本 skill,应拒绝,转 stock-stop-loss-decision)

### 这些情境的语言信号是什么?

- "该不该买""能买吗""适合买吗""能入手吗"
- "现在是不是高位""现在入场晚不晚"
- "我已经研究好了,你看怎么样""基本面很好,可以买吗"
- 出现"想抄底""想追涨"等情绪化表达,通常更需要本 skill 来过滤

### 和哪些相邻 skill 不同?

- **vs stock-stop-loss-decision**: 那个是已持仓,本 skill 是入场前
- **vs stock-position-sizing**: 那个是决定买多少,本 skill 是决定买不买
- **vs stock-market-timing**: 那个是大市判断(自上而下),本 skill 假设大市已通过(单独个股决策)

## E — 执行步骤 (Execution)

### 步骤 1: 大市环境检查

**动作**: 问用户——"当前大市是牛市/熊市/盘整?"。如果是熊市,**直接拒绝入场,转 stock-market-timing 学习**。

**完成标准**: 用户能清晰判断当前大市环境,且不是熊市。

### 步骤 2: 基础分析检查

**动作**: 问用户三个问题:
- "这家公司靠什么赚钱?"
- "未来 1-2 年的盈利增长动力是什么?"
- "管理层是否做了股票回购/分红等对股东友好的动作?"

**完成标准**: 用户能用 1-2 句话回答,且具体到可识别的事件/数字(不是"业绩好""成长股"这种模糊词)。

### 步骤 3: 阶段判断

**动作**: 教用户判断"临界点四阶段":
- **牛皮**:价格在支撑/阻力线间无方向震荡 → **不入场**
- **正常升势**: 一浪高过一浪 + 交易量配合 → **可入场**
- **疯狂**:所有人都在谈论这只股票 + 交易量暴增但股价不涨 → **不入场**
- **最后**:龙头股开始疲软 + 出现坏消息 → **不入场**

**完成标准**: 用户能正确识别目标股票当前阶段,且是"正常升势"。

### 步骤 4: 临界点识别

**动作**: 检查入场时机是否为临界点:
- 价格是否突破关键支撑/阻力线?
- 交易量是否在突破日放大?
- 市场情绪是否出现转折?(大众从"观望"转向"买入")

**完成标准**: 三个信号至少满足两个。

### 步骤 5: 止损位预设 + 仓位计算

**动作**: 在入场**之前**先定:
- 止损位(作者建议不超过总投资额 20%)
- 仓位 = (可承受亏损金额) / (入场价 - 止损价)

**完成标准**: 用户写下了"止损位 X 元,仓位 Y 股"的具体数字。

### 步骤 6: 分层下注

**动作**: 不要 all-in。建议: 20% 试仓 → 验证判断 → 60% 加码 → 100% 满仓。

**完成标准**: 用户的初次买入不超过目标仓位的 20-30%。

## B — 边界 (Boundary) ★

### 什么时候**不要**使用这个 skill

1. **用户没做基础分析时**: "我朋友说 X 会涨,我也想买"——本 skill 不应被触发,应先引导用户研究
2. **用户处于熊市**: 即使个股研究再充分,熊市胜算被 β 拖累,直接拒绝入场
3. **用户已经重仓**: 用户的"加仓"冲动往往是沉没成本,转 stock-stop-loss-decision
4. **用户被"赚钱"情绪驱动**: 当用户表达"再不上车来不及了",先转 stock-psychology-check

### 来自书中的警告

- **"已经涨了"不是入场理由**: 涨了 20% 的股票可能正在进入"疯狂阶段",这是反向信号
- **"基本面好"不是入场理由**: 王安电脑的教训——基本面+阶段+临界点必须三件套
- **"大市不好"是反向入场理由**: 作者原话"大市不好时,别买任何股票"

### 作者盲点

- **作者以美股经验为主**,未充分讨论 A 股的 T+1 制度、涨跌停板对入场时机的影响
- **作者未涉及 ETF/指数基金等现代工具**——本 skill 不适用于 ETF 决策(用更通用的"指数定投"决策框架)
- **作者对"基础分析"定义较为宽泛**——现代价值投资有更精细的财务指标(ROE/PE/PB 等),本 skill 不替代这些

### 与相邻 skill 的边界

- **stock-position-sizing**: 那个解决"买多少",本 skill 解决"买不买"
- **stock-stop-loss-decision**: 那个解决"亏了怎么办",本 skill 解决"赚的可能有多少"
- **stock-market-timing**: 那个解决"现在市场整体能不能买",本 skill 假设大市已通过
