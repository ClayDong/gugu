# 移动止损技术 (Trailing Stop)

## R — 原文 (Reading)

> 买入后,如果股票开始正常的升势,它应有一浪高过一浪的特点。你可以将止损点放在每个波浪的浪谷,随着波浪往上翻,你将卖点由A->B->C->D->E往上移。这样就能保证你不会在升势时过早离场。
>
> — 陈江挺,《炒股的智慧》第四章 第二节 何时卖股票

---

## I — 方法论骨架 (Interpretation)

移动止损是**"让利润奔跑"原则的具体执行机制**。它用"动态上移的止损点"解决了固定止盈的两难: 固定止盈太早(错过大趋势)、不设止盈太晚(回吐所有利润)。

**核心机制**:
- 初始: 在入场后设固定止损(支撑线/总资金 20%)
- 上升: 股价创新高 → 止损移到上一浪谷
- 加速: 趋势加速 → 可加码,但止损不动
- 触发: 跌破移动止损 → 自动出局

**为什么移动止损能解决"贪小便宜"问题**:
- 普通投资者: 赚 20% 跑 → 错过 200% 涨幅
- 移动止损: 涨 20% 时止损上移到成本 → 涨 50% 时止损上移到 20% 利润 → 涨 100% 时止损上移到 50% 利润 → 既保护了部分利润,又保留继续参与升势的可能

**为什么移动止损能解决"吃不得小亏"问题**:
- 移动止损是**机械规则**,不需要情绪判断
- 到点位就出,不需要"我觉得会反弹"

**波浪理论的应用**:
- 一浪高过一浪 = 上升趋势健康
- 浪谷是天然的"支撑"——跌破浪谷=趋势可能反转
- 浪谷递进 = 止损点递进

## A1 — 书中的应用 (Past Application)

### 案例 1: 利物莫的"安坐不动"
- **问题**: 利物莫的许多大盈利单都"看似应该获利了结",但他没动
- **方法论的使用**: 移动止损代替固定止盈——只要不跌破移动止损就持有
- **结论**: 利物莫成为本世纪初最著名的炒家
- **结果**: 验证了"移动止损"作为"让利润奔跑"工具的有效性

### 案例 2: 麦当劳 / 微软 / 王安电脑的高增长延长
- **问题**: 这些公司在盈利增长期持续上涨多年
- **方法论的使用**: 用"移动止损"代替"赚 20% 就跑"
- **结论**: 13 年百倍/4 年 11 倍/3 年 13 倍的回报
- **结果**: 验证了"让利润奔跑"在长期增长中的爆发力

### 案例 3: 永生集团"一灵散"(反例)
- **问题**: 永生集团"一灵散"从 5 元涨到 400 元
- **方法论的使用**: 没有在"疯狂阶段"用移动止损分批退
- **结论**: 信息反转 → 400 → 300 → 150 → 停牌
- **结果**: 从浮盈到停牌——警示: 即使有移动止损,在"疯狂阶段"也应主动减仓

### 用户会在什么情境下需要这个 skill?

1. **已浮盈**: "我赚了 20%,我想保护利润"
2. **想设止盈但又怕错失大涨**: "我设了 20% 止盈,但又怕后面还有大行情"

4. **规则化交易**: "我想要一个机械规则,不要凭感觉"
5. **已经手动设了固定止盈**: "我之前设了固定止盈,现在想改成动态"

## E — 执行步骤 (Execution)

### 步骤 1: 确认"正常升势"状态

**动作**: 在应用移动止损前,先确认股票处于"临界点四阶段"中的"正常升势"。
- 牛皮 → 不适用(本 skill 假设已确认升势)
- 正常升势 → 适用,继续
- 疯狂 → 分批退出,本 skill 退化为"分批减仓"
- 最后 → 立即清仓,本 skill 不适用

**完成标准**: 用户确认股票处于"正常升势"。

### 步骤 2: 识别当前浪谷

**动作**: 在当前走势中,识别最近的"浪谷"(价格回调的低点)。这将是新的止损位。

**完成标准**: 用户能说出"我的当前浪谷是 X 元"。

### 步骤 3: 上移止损位

**动作**: 把止损位从原始止损点,移到当前浪谷(略低于浪谷,如 3-5% 缓冲)。

**完成标准**: 用户写下"我的新止损位是 X 元"。

### 步骤 4: 设定再评估节奏

**动作**: 设定再评估的频率(每日/每周/每月)——每次价格创新高,重复步骤 2-3。

**完成标准**: 用户有固定的再评估时间表。

### 步骤 5: 触发时立即执行

**动作**: 跌破移动止损时,立即清仓,不要"再等等"。

**完成标准**: 用户接受"机械执行"原则。

### 步骤 6: 区分"正常回调"和"趋势反转"

**动作**: 不是每次跌破浪谷都是反转——可能是更深的回调。
- 跌破浪谷 1 次 → 警惕,观察
- 跌破浪谷 2 次 → 大概率反转,清仓
- 跌破浪谷 3 次 → 几乎确定反转,立即清仓

**完成标准**: 用户有明确的"几次跌破"规则。

## B — 边界 (Boundary) ★

### 什么时候**不要**使用本 skill

1. **用户已浮亏**: 用 stock-stop-loss-decision(本 skill 假设是浮盈状态)
2. **股票处于"疯狂"或"最后"阶段**: 应用"分批退出"或"立即清仓",不用本 skill
3. **用户想做长线价值投资**: 本 skill 是技术分析/趋势跟踪工具,不适用
4. **用户对波浪理论不熟悉**: 应先学习基础技术分析,本 skill 假设用户能识别浪谷
5. **震荡市/牛皮市**: 没有"一浪高过一浪",本 skill 不适用

### 来自书中的警告

- **不要试图寻找最高点**: 移动止损=中间一截,不是"逃顶"
- **"危险信号"三个**任一触发即减仓: 交易量暴增不涨/两天转头/坏消息
- **"疯狂阶段"必须主动分批退**: 移动止损在"疯狂阶段"失效(浪谷递进太慢)

### 作者盲点

- **作者未涉及现代算法交易的影响**: 程序化止损可能触发"止损踩踏",导致预设价位无法成交
- **作者未充分讨论"假突破"**: 浪谷跌破可能是诱空,不是反转
- **"再跌破几次"的规则因人而异**: 作者未给具体数字,需根据个人风险承受力调整

### 与相邻 skill 的边界

- **stock-profit-taking-decision**: 那个是"该不该卖"的决策,本 skill 是"怎么设止损"的技术
- **stock-stop-loss-decision**: 那个是浮亏时的固定止损,本 skill 是浮盈时的动态止损
- **stock-entry-decision**: 那个是入场前,本 skill 是入场后
